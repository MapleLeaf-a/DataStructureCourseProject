# AGENTS.md

## Build & Run

- **No build system** — compile manually:
  ```
  g++ code/cpp/main.cpp code/cpp/graph.cpp code/cpp/station.cpp code/cpp/menu.cpp -o metro.exe
  ```
- **Work in progress** — `path.cpp`/`path.h` are entirely commented out, `pathfinder.cpp`/`pathfinder.h` are empty. Do not include them in compilation.
- Run from repo root so `data/csv/` relative paths resolve correctly.
- Python: `python code/py/metro.py` (run from repo root, uses `../../data/txt/` and `../../data/csv/` relative paths).

## Encoding

- **All C++ source is GBK/GB2312 encoded** (Chinese comments and console output strings). Do not save as UTF-8 unless you also update all string literals.
- CSV files use `gbk`/`ANSI` encoding.
- Python `.txt` data files under `data/txt/` are UTF-8.

## Architecture

- **Python (data pipeline):** `code/py/metro.py` — reads Shanghai Metro API JSON from `data/txt/metro*.txt`, produces `data/csv/Station.csv` and `data/csv/Edge.csv`.
- **C++ (app):** Reads the two CSVs at runtime, provides a console menu for route planning.
- `graph.h/cpp` — adjacency list graph, Edge struct, CSV loader, `splitStr()` utility.
- `station.h/cpp` — Station struct, global `allStations` vector, status management.
- `menu.h/cpp` — console menu system. **Switch cases are empty stubs** — the menu structure exists but actions are not implemented.
- `path.h/cpp` — all commented out. Contains planned Dijkstra-based k-shortest paths + route dedup.
- `pathfinder.h/cpp` — empty placeholder.

## Menu Structure

```
1. Line/Station Info & Operations (8 sub-options — stubs)
2. Shortest Time Route (single / top-3 — stubs)
3. Minimum Transfer Route (single / top-3 — stubs)
4. Exit
```

## Key Conventions

- `using namespace std` in headers (do not change without refactoring all sources)
- `#ifndef` include guards (no `#pragma once`)
- Function naming: camelCase
- Chinese-language comments/UI strings throughout
- Edge times are random (2 or 3 minutes), generated by Python
